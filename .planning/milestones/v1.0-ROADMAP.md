# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-02-21
**Phases:** 1–6
**Total Plans:** 20

## Overview

Versioned contracts and formats; standalone runtime and control; SDK/CLI for device and pipeline config; documented standard path with recording; Hub-as-IDE; composable and extensible pipeline with published contracts and example outlet plugin.

## Phases

### Phase 1: Contracts and versioned formats
**Goal**: Config, profile, and stream semantics are versioned and documented so the same setup can be reproduced.
**Depends on**: Nothing (first phase)
**Requirements**: PATH-03, COMP-04, COMP-05
**Plans**: 3 plans

Plans:
- [x] 01-01 — Config format versioned and documented (COMP-05)
- [x] 01-02 — Profile format versioned; calibration/profile identity for reproducibility (COMP-05, PATH-03)
- [x] 01-03 — Stream semantics documented (consumption, timestamps, latest-sample) (COMP-04)

### Phase 2: Standalone runtime and control
**Goal**: User can run the decoder in the background and control it without the Hub GUI.
**Depends on**: Phase 1
**Requirements**: RUNT-01, RUNT-02, RUNT-03, COMP-03
**Plans**: 2 plans

Plans:
- [x] 02-01 — Standalone service with default control endpoint and startup docs (RUNT-01)
- [x] 02-02 — Control CLI (snapshot, set-output-enabled) and docs (RUNT-02, RUNT-03, COMP-03)

### Phase 3: SDK/CLI for device and pipeline config
**Goal**: Developer can drive device discovery and configure the signal/decoder pipeline via public API and CLI.
**Depends on**: Phase 2
**Requirements**: COMP-01, COMP-02
**Plans**: 2 plans

Plans:
- [x] 03-01 — SDK device discovery/connection API and CLI device list/connect (COMP-01)
- [x] 03-02 — Config YAML + docs, SDK config API, CLI config/pipeline subcommands (COMP-02)

### Phase 4: Standard path and recording
**Goal**: User has one coherent path from device to actions and can record/replay sessions for reproducibility.
**Depends on**: Phase 3
**Requirements**: PATH-01, PATH-02
**Plans**: 3 plans

Plans:
- [x] 04-01 — User guide with standard path section (device → decoder → run) (PATH-01)
- [x] 04-02 — Session recording (folder layout, tap task, control/CLI) (PATH-02)
- [x] 04-03 — Export to XDF and replay (virtual source, offline run, docs) (PATH-02)

### Phase 5: Hub-as-IDE
**Goal**: Hub is the IDE-like place for device setup, calibration, training, visualization, and one primary workflow.
**Depends on**: Phase 4
**Requirements**: HUB-01, HUB-02, HUB-03, HUB-04, HUB-05
**Plans**: 6 plans

## Milestone Summary

**Key Decisions:**
- Config format version and compatibility in config-format.md; N=2 previous versions supported
- Profile: calibration identity in metadata; N=2 reader compatibility
- Device API in neurohid-sdk; neurohid dispatches to neurohid-service for device|config|pipeline|control|daemon
- ConfigStore YAML/TOML by path extension; SDK config load/save; CLI config show/validate, pipeline run --dry-run
- Extension identity by name only; default discovery path config root + /extensions
- Loaded* wrappers hold libloading::Library + Box<dyn Trait>; snapshot exposes slot names

**Technical debt / deferred:**
- Plugin discovery/lifecycle (dynamic lib, subprocess, in-process) fully specified (EXT-04) — v2
